# JVM
* JVM : 자바 가상 머신(Java Virtual Machine)

## JVM 성능 분류
### 처리율
* 시스템이 수행 가능한 작업 비율을 나타내는 지표
* **일정 시간동안 완료한 작업 단위 수**를 표시(ex: 초당 처리 가능한 트랜잭션 수)
* 측정을 위해서는 플랫폼을 고려하고, 트랜잭션도 동일해야 한다.
* 실행 간 워크로드 역시 일정하게 유지해야 함. (워크로드 : 주어진 시간 내 처리해야 할 작업 할당량)

### 지연
* 1초에 100리터를 흘려보내는 수도관이 있다고 할때 이 수도관의 길이가 지연이랑 같다.
* 종단 시간이라고도 함.

### 용량
* 시스템이 보유한 작업 병렬성의 총량, 즉 시스템이 동시 처리가능한 작업 단위(트랜잭션) 개수.
* 보통 **처리율 또는 지연 값을 전제로 가능한 처리량으로 표시*.*

### 사용률
* 시스템 리소스를 얼마나 효율적으로 활용하는가에 대한 지표
    * CPU나 메모리를 계속 IDLE 상태로 둘수록 사용률이 낮아짐.
* 워크로드에 따라서 리소스별로 들쑥날쑥함.
    * 계산 직얍적 워크로드(그래픽, 암호화 처리)는 CPU 사용률이 100%에 달하지만 메모리 사용률은 낮다.
    
### 효율
* 처리율을 리소스 사용률로 나눈 값.
* 같은 처리율에 리소스 사용률이 높을수록 효율이 떨어진다.

### 확장성
* **리소스 추가에 따른 처리율 변화**는 시스템/애플리케이션의 확장성을 가늠하는 척도.
* 리소스를 추가 한 만큼 처리율이 그만큼 증가(리소스 2배 -> 처리율 2배)한다면, 완벽한 선형 확장성을 가짐.
* 하지만 선형 확장성은 현실적으로는 매우 어려운 일.

### 저하
* 시스템에 많은 부하를 받으면(사용률이 높아지면) 처리율이 더는 늘어나지 않는 상태가 발생함.
* 이런 현상을 저하라고 함.
