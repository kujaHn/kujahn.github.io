# 람다
* **메소드로 전달할 수 있는 익명 함수를 단순화한 것**
    * 함수 : 메소드와 달리 특정 클래스에 종속되지 않음. (파라미터 리스트, 바디, 반환형식, 예외 리스트는 가질 수 있다.)
    
## 람다의 구성 요소
* 람다는 **파라미터, 화살표, 바디**로 이루어져 있다.  
  <img src="/image/JAVA8/ch3/ch3_1.png" alt="람다의 구성 요소.png" class="align-center">
* **람다 바디** : 반환값에 해당하는 표현식.
  ```
  (parameters) -> expression      // 표현식 스타일
  (parameters) -> {statements;}   // 블록 스타일 (구문 사용)
  ```
  * 구문과 표현식의 차이를 모르겠다면 [다음](!주소쓰기)을 참고하자.

## 람다 작성하기
### Basic
* **람다 표현식은 `함수형 인터페이스`에서만 사용이 가능하다.**

  ### 함수형 인터페이스
  * **오직 하나의 추상 메소드만을 지정하는 인터페이스를 함수형 인터페이스**라 한다.
    * 2장에서 만든 `Predicate<T>` 인터페이스가 함수형 인터페이스의 대표적 예시이다.
  
  * 번외로 함수형 인터페이스를 가르키는 `@FunctionalInterface` 어노테이션이 있다.
    * 실제로 함수형 인터페이스가 아니라면 컴파일러 에러를 발생시킨다.
    * ex: 추상메소드가 1개 이상인 경우 `"Multiple non-overriding abstract methods found in Interface"` 에러 발생.

  ### 함수 디스크립터
  * 함수형 인터페이스의 추상 메소드 시그니처는 **람다 표현식의 시그니처**를 가르킨다.
  * **람다 표현식의 시그니처를 서술하는 메소드를 함수 디스크럽터 라고 부른다.**
    * **시그니처 : 람다(메소드) 파라미터 + 람다 바디(메소드 리턴)**
  
### 람다 규칙
*
  ### 1st. return을 명시적으로 사용하지 않아도 된다
  * **return을 명시적으로 사용해도, 사용하지 않아도 상관 없다.**
      ```java
      // 두 코드 모두 문자열 Hello를 반환.
      () -> "Hello"
      () -> {return "Hello";}
      ```
    
  ### 2nd. 여러 행의 코드를 가질 수 있다.
  * 블록 안에 여러 행의 코드를 가질 수 있다.
      ```java
      (int x, int y) -> {
          System.out.println("합의 결과");
          System.out.println("x + y = " + x + y);
      }
      ```

## 학생 기록부 V6
* 이전 장의 `predicateFilterV4` 메소드를 통해 배운것을 정리하자.
    ```java
    public List<Apple> maleStudents = predicateFilterV4(students, new StudentPredicate() {
        public boolean test(Student student) {
            return student.getGender().equals(Gender.Male);
        }
    });
    ```
### 1st. 람다로 변환하고 싶은 요소가 함수형 인터페이스인가?
* 다시 말하자면 함수형 인터페이스는 **단 하나의 추상 메소드만을 정의하는 인터페이스라고 했다.**
  ```java
  interface StudentPredicate {
      boolean test(Student student);
  }
  ```
  * `StudentPredicate`는 함수형 인터페이스가 맞으므로, 람다로 변환이 가능하다.

### 2nd. 람다의 구성 요소로 변경.
* **메소드의 파라미터 부분을 람다의 파라미터**에, **메소드의 리턴을 람다의 바디에 매칭**시킨다.
  <img src="/image/JAVA8/ch3/ch3_2.png" alt="람다 매핑.png" class="align-center">
  
* 람다의 구성 요소에 맞게 변환을 하면 다음과 같은 람다식이 나온다.
    ```java
    public List<Student> maleStudents = predicateFilterV6(students, (Student s) -> s.getGender().equals(Gender.Male));
    ```

## 요약
* 람다 표현식은 익명함수의 일종!
    * **메소드 명이 없으며 `파라미터 리스트`, `바디`, `반환 형식`을 가지며 `예외`를 던질 수 있다.**
* **`함수형 인터페이스`는 하나의 추상 메소드만을 정의하는 인터페이스**
    * 오직 함수형 인터페이스의 추상 메소드(익명 함수)만 람다 표현식으로 대체할 수 있다.
    * **람다 표현식 전체가 함수형 인터페이스의 인스턴스토 취급된다.**
